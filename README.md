# ホットラインAIエージェント ローカルプロトタイプ

GitHub Pages から切り離した、ローカル起動用の独立プロトタイプです。
React、Vite、Babel、CDN、npm install は不要です。

## 起動方法

### 方法1：もっとも簡単

`index.html` をブラウザで開いてください。

### 方法2：ローカルサーバーで起動

Node.js が入っている環境で、以下を実行してください。

```bash
npm start
```

起動後、ブラウザで以下を開きます。

```text
http://localhost:3000
```

Windows の場合は `start-windows.bat` をダブルクリックしても起動できます。
Mac / Linux の場合は以下です。

```bash
./start-mac-linux.sh
```

## この版の仕様

- 現行UI基盤を維持したチャットボットUI
- Markdown要件定義に基づく4カテゴリ構成
  - Ⅰ. 調剤ミス
  - Ⅱ. 個人情報漏洩
  - Ⅲ. 管理薬剤の紛失
  - Ⅳ. その他トラブル
- 本部区分、店番、店舗名、店番連動の営業部・地区名・PSV名表示
- 調剤ミス14分類と、分類別の動的質問分岐
- 全カテゴリ後の速報判断項目
- 速報対象の自動表示
- AI要約確認フロー
- 項目単位の修正フロー
- 電話相談へのフォールバック
- 音声による選択・テキスト入力
- Slack送信プレビューのブロック整形
- Slack通知、管理表転記はモック表示

## 注意

このプロトタイプは外部送信を行いません。
入力内容はブラウザ画面上でのみ扱われます。
実運用でSlack、Google Sheets、LLM APIと接続する場合は、バックエンド実装と認証・権限設計が別途必要です。

音声入力は、利用するブラウザの音声認識機能に依存します。
ブラウザ側でマイク利用の許可が必要です。
利用できない場合でも、通常のボタン選択・テキスト入力で同じフローを完了できます。

## v5 UIUX変更点

- 画面上部60%をチャットボット会話エリア、下部40%をユーザー選択・入力エリアに固定しました。
- 会話エリアと入力エリアは、それぞれ独立して縦スクロールできます。
- ヘッダー背景をグリーン、文字を白に変更しました。
- AIの案内文を「安心してください」「それはお困りですね」など、より温かく寄り添う口調に調整しました。

## v6 改訂点

- 添付のMarkdown要件定義を `requirements_hotline_ai_agent.md` として同梱しました。
- UI基盤は維持し、質問定義を新要件へ更新しました。
- 最終確認・受付完了画面に、見出し・項目・改行を整理したSlack投稿プレビューを追加しました。
- ブラウザの音声認識機能に対応している場合、選択肢は音声で選択、自由入力欄は音声で入力できます。


## v7 スマホ入力向け変更点

- アプリ全体を「ヘッダー / 本体 / フッター」の3領域に分け、回答操作ボタンが画面下で見切れにくい構成にしました。
- チャットスペースと入力スペースは、画面内でそれぞれ縦スクロールできます。
- `viewport-fit=cover` と safe-area 対応を追加し、スマホのノッチ・ホームインジケータ周辺でも操作部が隠れにくいようにしました。
- 画面拡大、ダブルタップ拡大、ピンチ拡大、Ctrl+ホイール拡大を抑止する処理を追加しました。
- 横スクロールが出にくいよう、全体幅を `100vw` に制限し、長文・Slackプレビューは折り返し表示にしました。
- 音声入力UIを、説明カード形式から控えめな丸いマイクボタンのみに変更しました。

## v8 レスポンシブ対応

- スマホ表示はv7のUIを維持し、上からヘッダー、チャット、入力、フッターの縦構成のままです。
- PC表示では、960px以上の画面幅で左をチャットウインドウ、右を入力ウインドウに切り替えます。
- PC表示でも、チャット欄と入力欄はそれぞれ独立して縦スクロールできます。
- 横スクロール抑止、画面拡大抑止、マイクボタンのみの音声入力UIは継続しています。
