# ホットラインAIエージェント ローカルプロトタイプ

GitHub Pages から切り離した、ローカル起動用の独立プロトタイプです。
React、Vite、Babel、CDN、npm install は不要です。

## 起動方法

`index.html` をブラウザで開いてください。

## ファイル構成

```text
sugi_hotline_system/
├─ index.html
├─ style.css
├─ app.js
├─ requirements_hotline_ai_agent.md
├─ project_overview_detail.txt
└─ README.md
```

## 実装済み機能

### ① カレンダー日付選択（投薬日・発覚日）

「投薬日」「発覚日」「発生日」などの日付質問では、**📅 カレンダーUIと相対日付ボタンの両方**から選択できます。

- **📅 カレンダーから日付を選択**セクション（青枠）が相対日付ボタンの上に常に表示される
- `<input type="date">` のカレンダーで日付を選ぶと、**決定ボタンを押す前にプレビュー表示**で選択日を確認できる
- 「この日付で決定する ›」ボタンを押して確定（日付未選択時はボタン非活性）
- 従来の「本日」「昨日」「2日前…」ボタンからも選択可能
- **どちらから選んでも `yyyy/mm/dd` 形式で回答を収集**
- 「それ以前」「不明」ボタンはそのまま使用可能
- カレンダーは未来日を選択不可（`max=today` 設定）

---

### ② ファイル・写真の添付（フッター配置・モック）

フッターの「📎 ファイル選択」ボタンからファイルを添付できます。

- フッターに**3ボタン横並び**：**回答を修正 / 電話で相談 / ファイル選択**
- タップしてファイル（画像・PDF・Office文書・CSV等）を選択
- 選択後、フッター下部にファイル名チップが表示（削除ボタン付き）
- **現在はモック**（実際のアップロードは行わない）
- ファイル名は回答ログに記録される

---

### ③ 回答ログの収集

送信完了後、回答内容をブラウザの `localStorage` に**自動保存**します。  
バックグラウンドでのWebhook送信にも対応（設定が必要）。

**ログの取り出し方：**

- 完了画面の「回答ログをダウンロード（JSON）」ボタン

**バックグラウンド送信（オプション）：**

`app.js` の `LOG_SEND_CONFIG` を編集してWebhookエンドポイントを設定すると、送信完了時にバックグラウンドでPOSTリクエストを送信します。

```javascript
const LOG_SEND_CONFIG = {
  method: "webhook",            // "webhook" | "none"
  endpoint: "https://formspree.io/f/XXXXXXXX"  // 送信先URL
};
```

**ログ形式（JSON）：**

```json
{
  "id": "HL-20260518-1234",
  "timestamp": "2026/05/18 14:30",
  "startedAt": "2026/05/18 14:00",
  "answers": { "category": "Ⅰ. 調剤ミス", "dose_date": "2026/05/17" },
  "attachedFiles": ["receipt.pdf", "photo.jpg"]
}
```

> **GitHub Pages（静的サイト）の制限について：**  
> GitHub Pages はサーバーレス静的ホスティングのため、サーバーへのログ送信はデフォルト無効です。  
> ログは各端末の `localStorage` に蓄積されます。複数端末のログを集約するには  
> 各端末でダウンロードした JSON を手動で統合してください。  
> Webhook送信を使う場合は、Formspree・Make（Integromat）・Zapier 等のサービスと組み合わせてください。

### ④ チャット／回答パネルのサイズ調整（モバイルのみ）

モバイル画面（幅 960px 未満）で、チャットエリアと回答エリアの**上下比率を調整**できます。

| 項目 | 内容 |
|------|------|
| 初期比率 | 1:1（上下均等） |
| 調整方法（タッチ） | チャットと回答パネルの間のハンドルバーを指でドラッグ |
| 調整方法（マウス） | ハンドルバーをマウスでドラッグ（PC幅未満の場合） |
| 調整方法（キーボード） | ハンドルにフォーカス後、↑↓ キーで操作可能 |
| 調整範囲 | 20%〜80% |
| PC表示 | 960px 以上は左右2カラム固定、ハンドル非表示 |

---

### ⑤ 文字サイズ 5段階（ヘッダー横並びボタン）

ヘッダー下段に **「最小」「小」「標準」「大」「最大」の5つのボタン**を横並びで表示。

| 段階 | ラベル | PC基準 | モバイル基準 |
|------|--------|--------|-------------|
| 1    | 最小   | 14px   | 13px        |
| 2    | 小     | 16px   | 15px        |
| 3    | **標準**（デフォルト） | 20px | 18px |
| 4    | 大     | 23px   | 20px        |
| 5    | 最大   | 27px   | 23px        |

- 現在選択中のサイズは**白背景・緑文字**でハイライト表示
- 選択したサイズは `localStorage` に保存され、次回起動時に引き継がれる

---

### ⑥ 音声入力（マイクボタン）

テキスト入力欄の横に表示される **🎙️ マイクボタン**から音声で回答を入力できます。

- 目立つ円形デザイン・グリーンボーダー
- 録音中はボタンが緑に変わり、**パルスアニメーション**で状態を明示
- ブラウザの音声認識機能に依存するため、非対応環境ではボタン非表示
- 音声認識できない場合でも、通常のボタン選択・テキスト入力で同じフローを完了できる

## 注意

このプロトタイプはデフォルトで外部送信を行いません。
入力内容はブラウザ画面上（および `localStorage`）でのみ扱われます。

実運用でSlack、Google Sheets、LLM APIと接続する場合は、バックエンド実装、認証、権限管理、監査ログ、障害時再送設計が別途必要です。

音声入力はブラウザの音声認識機能に依存します。
利用できない場合でも、通常のボタン選択・テキスト入力で同じフローを完了できます。
