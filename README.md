# ホットラインAIエージェント ローカルプロトタイプ

GitHub Pages から切り離した、ローカル起動用の独立プロトタイプです。
React、Vite、Babel、CDN、npm install は不要です。

## 起動方法

`index.html` をブラウザで開いてください。

## ファイル構成

```text
sugi_hotline_system/
├─ index.html
├─ style.css
├─ app.js
├─ requirements_hotline_ai_agent.md
├─ project_overview_detail.txt
└─ README.md
```

## この版の位置づけ

直近のレスポンシブ対応済みプロトタイプをベースに、以下の機能を追加した版です。

## 機能一覧

### 基本 UI（継続）

- スマホ：ヘッダー、チャット、入力、フッターの縦構成
- PC：960px以上で左チャット、右入力の2カラム
- 横スクロール抑止 / 縦スクロール許可 / 画面拡大抑止
- 控えめな丸いマイクボタンによる音声入力
- Slack送信プレビューのブロック整形

---

### ① カレンダー日付選択（投薬日・発覚日）

「投薬日」「発覚日」「発生日」などの日付入力質問で、**カレンダーUI**と**相対日付ボタン**の両方から日付を選択できます。

- カレンダー（`<input type="date">`）で日付を選び「決定」ボタンを押す
- または従来の「本日」「昨日」「2日前…」ボタンから選択
- **どちらから選んでも `yyyy/mm/dd` 形式で回答を収集します**
- 「それ以前」「不明」ボタンはそのまま利用可能

---

### ② 回答ログの収集

送信完了後、回答内容をブラウザの `localStorage` に自動保存します。

**ログの取り出し方：**

- 完了画面の「回答ログをダウンロード（JSON）」ボタン
- フッターの「ログ保存」ボタン

**ログ形式（JSON）：**

```json
{
  "id": "HL-20260518-1234",
  "timestamp": "2026/05/18 14:30",
  "startedAt": "2026/05/18 14:00",
  "answers": { "category": "Ⅰ. 調剤ミス", "dose_date": "2026/05/17", ... },
  "attachedFiles": ["receipt.pdf", "photo.jpg"]
}
```

> **GitHub Pages（静的サイト）の制限について：**  
> GitHub Pages はサーバーレス静的ホスティングのため、ブラウザ外へのログ送信は行いません。  
> 各端末の localStorage にログが蓄積されます。複数端末のログを集約するには、各端末でダウンロードしたJSONを手動で統合してください。  
> 実運用でサーバーサイドへのログ収集が必要な場合は、バックエンドAPIまたは Google Forms / Supabase 等のサービスとの接続が別途必要です。

---

### ③ ファイル・写真の添付（モック）

回答パネル上部に**常設のファイル添付エリア**があります。

- 「📎 ファイル・写真を添付（任意）」ラベルをタップ/クリックしてファイルを選択
- 画像・PDF・Office文書・CSV 等に対応
- 添付したファイル名はチップ形式で表示（削除ボタン付き）
- 「最初から」リセット時は添付情報もクリア
- **現在はモック（実際のファイルアップロードは行いません）**
- ログ収集時は **ファイル名** のみを記録します

---

### ④ チャット/回答パネルのサイズ調整（モバイル）

モバイル画面（幅960px未満）で、チャットエリアと回答エリアの**上下比率をドラッグで調整**できます。

- チャットと回答パネルの境目にあるハンドル（グレーのバー）をドラッグ
- 上下比率の範囲：チャット20%〜80%
- キーボードフォーカス時は ↑↓ キーでも調整可能
- PC表示（960px以上）ではハンドルは非表示（左右2カラム固定）

---

### ⑤ 文字サイズ 5段階調整

ヘッダーの「**文字：標準**」ボタンを押すたびに文字サイズが切り替わります。

| 段階 | ラベル | PC基準 | モバイル基準 |
|------|--------|--------|-------------|
| 1    | 最小   | 14px   | 13px        |
| 2    | 小     | 16px   | 15px        |
| 3    | **標準**（デフォルト） | 20px | 18px |
| 4    | 大     | 23px   | 20px        |
| 5    | 最大   | 27px   | 23px        |

- 選択した文字サイズは `localStorage` に保存され、次回起動時に引き継がれます
- ボタンには現在のサイズ名が表示されます（例：「文字：大」）

---

## 注意

このプロトタイプは外部送信を行いません。
入力内容はブラウザ画面上（および localStorage）でのみ扱われます。

実運用でSlack、Google Sheets、LLM APIと接続する場合は、バックエンド実装、認証、権限管理、監査ログ、障害時再送設計が別途必要です。

音声入力はブラウザの音声認識機能に依存します。
利用できない場合でも、通常のボタン選択・テキスト入力で同じフローを完了できます。
